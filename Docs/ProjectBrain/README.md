# Project Brain Index

Project Brain docs are compact topic summaries for Codex context loading. They organize the current project state without replacing the source of truth.

`Docs\Project_Status.md` remains the authoritative source for current release status, route logic, blocking rules, active issues, confirmed behavior, and next recommended tasks. If a Project Brain note conflicts with `Docs\Project_Status.md`, use `Docs\Project_Status.md`.

## Files

- `Current_State.md`: short project summary, stable behavior, current development focus, and active known issues.
- `Goblin_Tracker.md`: automatic counting, retired manual count behavior, countable and blocked areas, suppression rules, test status, and remaining validation.
- `Routes_And_Areas.md`: route order, area grouping, blocked transitions, countability, and special route/count cases.
- `Debug_And_Testing.md`: debug package workflow, replay status, validation commands, live-test flow, and useful review evidence.
- `Release_And_Build.md`: build, publish, installer, version, and release note workflow summarized from existing release docs.
- `Codex_Workflow.md`: working rules for Codex sessions on this repo.

## Related Source Docs

- `Docs\Project_Status.md`: main source of truth.
- `Docs\TODO.md`: open validation items and future work.
- `README.md`: high-level user-facing behavior and build/release instructions.
- `AGENTS.md`: required agent workflow and repository rules.
- `Docs\Release_Checklist.md` and `Docs\Release_v1.5.md`: release-specific validation and notes.
